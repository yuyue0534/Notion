**xiaoya-alist** 是一个在中文 NAS（网络附属存储）和家庭媒体中心社区非常火爆的开源项目。

简单来说，它是一个**“一键部署脚本集合”**，旨在帮助用户极其轻松地搭建基于 **AList** 的阿里云盘聚合站，并进一步整合 **Emby/Jellyfin** 媒体服务器和 **TVBox** 播放源，实现“海量云端资源本地化播放”的家庭影院方案。

该项目由开发者 **DDS-Derek** 维护（注意：核心资源提供方是“小雅”，但此 GitHub 仓库是部署工具的集合），它极大地降低了普通用户搭建复杂媒体服务器的门槛。

---

### 1. 核心功能：它能做什么？

这个项目不仅仅是一个软件，而是一套**自动化运维脚本**，主要解决以下三个层面的问题：

#### 📦 A. 快速部署“小雅 AList”
*   **背景**：AList 是一个支持多种存储（包括阿里云盘）的文件列表程序。而“小雅”是一个大神维护的共享资源库，聚合了数百个阿里云盘分享链接（包含海量的电影、电视剧、动漫、纪录片，总量达几十 TB 甚至 PB 级）。
*   **作用**：`xiaoya-alist` 脚本可以一键在你的 NAS 或服务器上部署好 AList，并自动配置好连接“小雅”资源库的参数。你不需要自己去搜集资源，部署完就直接拥有海量影视库。

#### 🎬 B. 搭建“小雅 Emby/Jellyfin”媒体服务器
*   **痛点**：直接用 AList 看视频虽然可以，但没有海报墙、没有剧情简介、无法记录观看进度，体验像在看文件夹。
*   **作用**：脚本可以一键部署 Emby 或 Jellyfin 容器，并自动从云端同步元数据（海报、简介、演员表等）。
*   **效果**：让你拥有和 Netflix、爱奇艺一样的精美海报墙界面，但内容却是云端的免费资源。且**不占用本地硬盘空间**（元数据仅需约 100GB-200GB，视频流直接走云端）。

#### 📺 C. 生成 TVBox 接口
*   **作用**：为电视端的 TVBox、影视仓等播放器生成专用的数据源接口。
*   **效果**：让你在安卓电视盒子上也能直接通过 TVBox App 观看小雅聚合的资源，支持搜索、分类和高清播放。

---

### 2. 为什么它这么火？（核心优势）

1.  **极低的空间成本**：
    *   传统 NAS 玩影音需要购买大容量硬盘（4TB、8TB...）。
    *   使用 `xiaoya-alist` 方案，视频文件存在阿里云盘（云端），本地只存储**元数据**（海报、简介信息）。搭建一个拥有数万部影片的海报墙，本地仅需 **160GB - 200GB** 左右的存储空间。

2.  **一键自动化（傻瓜式）**：
    *   原本需要手动配置 Docker、挂载卷、获取 Token、设置定时任务、下载元数据等复杂步骤。
    *   该项目提供了一条命令（`bash -c "$(curl ...)"`），全自动完成所有配置，甚至包括自动更新。

3.  **资源极其丰富且更新快**：
    *   依托“小雅”社群的维护，资源库几乎涵盖了全网热门的影视内容，且更新速度极快（往往比流媒体平台还快）。

4.  **多平台支持**：
    *   支持群晖 (Synology)、威联通 (QNAP)、极空间、绿联等成品 NAS。
    *   支持 Linux 服务器、Docker 环境，甚至部分 Android 设备。

---

### 3. 工作原理简述

1.  **资源层**：资源实际存储在“小雅”维护的多个阿里云盘账号中。
2.  **代理层 (AList)**：你的设备上运行 AList，通过 Token 连接到这些阿里云盘，将云端文件“映射”为本地网络路径。
3.  **展示层 (Emby/Jellyfin)**：媒体服务器扫描 AList 挂载的路径，下载元数据（海报等）到本地缓存，生成精美的海报墙。
4.  **播放层**：当你点击播放时，视频流直接从阿里云盘流经你的网络到达播放设备（类似在线流媒体，但拥有本地管理的体验）。

---

### 4. 如何开始？（简易流程）

如果你有一个安装了 Docker 的 NAS 或 Linux 服务器，通常只需以下几步：

1.  **准备环境**：确保已安装 Docker 和 Docker Compose。
2.  **获取 Token**：你需要一个阿里云盘账号（建议注册小号），获取 `refresh_token`（项目文档中有详细教程，通常通过扫码获取）。
3.  **执行一键脚本**：
    在终端运行项目提供的命令，例如：
    ```bash
    bash -c "$(curl --insecure -fsSL https://ddsrem.com/xiaoya_install.sh)"
    ```
    *(注：具体命令请以 GitHub 仓库 README 为准，因为地址可能会更新)*
4.  **跟随向导**：脚本会交互式地询问你一些配置（如存储路径、端口号、Token 等），按提示输入即可。
5.  **等待同步**：首次启动后，系统会自动下载元数据（可能需要几十分钟到几小时，取决于网络和选择的内容量）。
6.  **享受观影**：通过浏览器访问 Emby 地址，或在电视上配置 TVBox 接口。

---

### 5. 注意事项与风险

*   **依赖阿里云盘政策**：该方案的核心依赖于阿里云盘的第三方接口政策。如果阿里官方严格限制第三方流量或封禁账号，服务可能会受影响（虽然小雅团队通常有多个账号轮换，稳定性较高，但风险依然存在）。
*   **网络要求**：播放画质（尤其是 4K）取决于你的网络带宽以及阿里云盘对该地区的限速情况。非会员账号在某些时段可能会被限速。
*   **隐私问题**：虽然是通过 AList 代理，但理论上你访问的资源列表和播放记录会经过小雅的服务器进行元数据匹配。介意隐私的用户需自行权衡（这也是为什么很多人选择自建 Emby 而不是直接用别人的接口）。
*   **并非完全离线**：这本质上是一个“高级在线播放”方案，断网无法观看。

### 总结

**xiaoya-alist** 是目前中文圈**性价比最高、上手最简单**的家庭影院解决方案之一。它让没有大硬盘、不懂复杂技术的普通用户，也能轻松搭建起拥有海量 4K 资源的海报墙媒体库。

如果你想在 NAS 上玩影音，但又不想花几千块买硬盘，或者不想花时间找资源，这个项目是首选。
